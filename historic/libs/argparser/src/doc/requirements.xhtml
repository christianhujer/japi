JAPI Lib ArgParser - Requirements

  • REQ001: The library MUST provide option parsing. Options start with a "-"-character. Every option has a unique name. A name that consists of a single letter is called short option. A name that consists of more than one letter is called long option.
  • REQ002: Optional options MUST be supported. It MUST be possible to declare an option as optional. If an optional option is omitted, the library MUST simply ignore that option. If an optional option is specified, the library MUST handle the option properly. User view: An optional option may be omitted.
  • REQ003: Mandatory options MUST be supported. It MUST be possible to declare an option as mandatory. If a mandatory option is omitted, the library MUST NOT process the command. If a mandatory option is specified, the library MUST handle the option properly. User view: A mandatory option must not be omitted.
  • REQ004: Terminating options MUST be supported. It MUST be possible to declare an option as terminating. A terminating option may be omitted.
  • REQ005: The library MUST support i18n/l10n.
  • REQ006: Options with names which consist of non-ASCII characters MUST be rejected.
  • REQ007: It SHOULD be possible to concatenate multiple short options in a single option that looks like a long option starting with one dash.
  • REQ008: Options that start with two dashs MUST always be treated as long options.
  • REQ009: If an option that starts with a single dash is not recognized as a long option it SHOULD be treated as concatenated short options. If treating it as concatenated short options fails, the option SHOULD be reported as unrecognized long option.
  • REQ00A: It MUST be possible to stop further arguments from being parsed as options using the special option "--".
  • REQ00B: It MUST be possible to declare an option as obsolete. An obsolete option will usually perform only an empty operation. The user will be warned if she uses an obsolete option.
  • REQ00C: It MUST be possible to specify long options using -W.
  • REQ00D: It MUST be possible to specify options in a file by prefixing the filename with '@'.
